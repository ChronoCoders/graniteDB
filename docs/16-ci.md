# 16. CI Automation

GraniteDB uses CI to continuously enforce correctness gates.

## 16.1 Required Gates

- formatting (`cargo fmt --check`)
- unit + integration tests (`cargo test`)
- clippy (`cargo clippy --all-targets --all-features -- -D warnings`)

## 16.2 Loom Gate

Loom tests run under the `loom` feature:

```bash
cargo test --test loom --features loom
```

## 16.3 Miri Gate

Miri runs on a schedule and can be triggered manually:

```bash
cargo +nightly miri setup
cargo +nightly miri test
```
