# 16. CI Automation

GraniteDB uses CI to continuously enforce correctness gates.

## 16.1 Required Gates

- formatting (`cargo fmt --check`)
- unit + integration tests (`cargo test`)
- clippy (`cargo clippy --all-targets --all-features -- -D warnings`)

## 16.2 Loom Gate

Loom tests run under the `loom` feature:

```bash
cargo test --test loom --features loom
```

## 16.3 Miri Gate

Miri runs on a schedule and can be triggered manually:

```bash
cargo +nightly miri setup
cargo +nightly miri test
```

## 16.4 Fuzz Gate (Smoke)

Fuzzing runs on a schedule and can be triggered manually. CI uses a bounded “smoke” run count.

Prereqs:

```bash
cargo install cargo-fuzz
```

Run:

```bash
cargo fuzz run fuzz_wal -- -runs=2000
```

## 16.5 Sanitizers Gate

Sanitizers run on a schedule and can be triggered manually. CI uses nightly and `-Zbuild-std`.

ASan:

```bash
RUSTFLAGS="-Zsanitizer=address" cargo +nightly test -Zbuild-std
```

TSan:

```bash
RUSTFLAGS="-Zsanitizer=thread" cargo +nightly test -Zbuild-std
```
